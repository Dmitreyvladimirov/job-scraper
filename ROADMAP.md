# Job Scraper — Roadmap

### Инфраструктура
- [x] Railway deployment: cron job пн–пт 4×/день
- [x] `run.sh` entrypoint, `railway.toml`
- [x] SQLite логирование прогонов и статистики (`db.py`)

### Фильтрация
- [x] Фильтр по роли (PM-ключевые слова в title)
- [x] Фильтр по локации (Remote / Israel / EMEA, исключения US-only и т.п.)
- [x] Фильтр по дате публикации (MAX_JOB_AGE_DAYS = 14)
- [x] Языковой фильтр: EN/ES/RU проходят, DE/FR/NL блокируются (с исключением для русскоязычных требований)

- [x] Cross-run дедупликация по (company, title) через Notion seen_keys
- [x] normalize_job_key(): fuzzy-матчинг, стрипает Inc/Ltd/GmbH и пунктуацию

### ATS-скоринг
- [x] GPT-4o-mini, 4 измерения: Role / Domain / Keywords / Location
- [x] Domain Fit разделён на Value (0–15) + Experience (0–15)
- [x] Penalty −15 за жёсткие требования по домену
- [x] Must Have / Nice to Have для keyword overlap
- [x] Рубрика синхронизирована с ResumeBuilder

### URL-обогащение
- [x] Поиск прямой ссылки через Greenhouse / Lever / Ashby API

- [x] Предупреждение в Notion если прямая ссылка не найдена (неполное описание)

### Notion

## В планах

### Высокий приоритет

#### Прямые источники по компаниям
Опрашивать Greenhouse/Lever/Ashby напрямую для списка 30–50 целевых компаний.
Даёт вакансии которые не попадают на агрегаторы вообще.
- Составить список целевых компаний (Israeli tech, remote-first стартапы, AI-компании)
- Написать `sources/greenhouse_direct.py`, `sources/lever_direct.py`
- Добавить в pipeline наравне с другими источниками

#### Аналитический дашборд
FastAPI + Chart.js на Railway (отдельный сервис, тот же репо).
- KPI: всего прогонов, qualified, GPT-вызовы
- Графики: qualified/fetched по дням, воронка фильтров, распределение ATS-скоров, эффективность источников
- Таблица последних прогонов
- Защита по токену (`?token=xxx`)
- Статус: код написан (`dashboard.py`), деплой не завершён (Railway конфигурация)

### Средний приоритет

#### Feedback loop
Когда статус карточки в Notion меняется на "applied" или "interview" — бот должен знать.
Позволит анализировать: какие паттерны у вакансий куда реально откликаешься.
- Webhook или polling Notion API
- Хранить историю статусов в SQLite
- Показывать в дашборде: conversion rate по источникам и доменам

#### Еженедельный дайджест
Вместо 4 уведомлений в день с 0–2 вакансиями — один digest в пятницу.
Топ-5 за неделю, статистика прогонов, сравнение с прошлой неделей.

#### WeWorkRemotely и Arbeitnow: fetch full JD
Аналогично RemoteOK — после обогащения URL загружать полный JD из прямого источника.
Сейчас это сделано только для RemoteOK.

### Низкий приоритет / идеи

#### Company research при высоком ATS
Когда вакансия проходит с score ≥ 75 — автоматически добавлять в Notion 2–3 факта о компании
(размер, стадия, продукт). Для лучшего контекста перед откликом.

#### Salary tracking
Если в JD есть диапазон зарплат — парсить и хранить как отдельное поле в Notion.
Позволит фильтровать по уровню компенсации.

#### Навыковый gap-анализ
По отклонённым вакансиям (rejected) — какие keywords стабильно в missed?
Показывать в дашборде: "Вот 5 скиллов которых не хватает чаще всего".
